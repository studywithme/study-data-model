c) OLTP는 복잡한 구조, OLAP는 단순한 구조

- 각 시스템의 목적에 따라 최적화 방향이 다름<br>
- 스타 스키마는 OLAP의 대표적인 비정규화 모델
</details>

### 6. 표준화 문제 (30점)

**상황**: 다음과 같은 기존 테이블들이 있습니다. 표준화 관점에서 문제점을 찾고 개선안을 제시하세요.

```sql
-- 기존 테이블들
CREATE TABLE TBL_CUST (
    C_ID VARCHAR(10),
    C_NM VARCHAR(50),
    C_PHONE VARCHAR(15),
    REG_DT DATE
);

CREATE TABLE ORDER_INFO (
    ORDER_NO VARCHAR(20),
    CUSTOMER_ID VARCHAR(10),
    ORD_DATE DATE,
    TOTAL_AMT NUMBER
);

CREATE TABLE PROD (
    PRODUCT_CODE VARCHAR(15),
    PROD_NAME VARCHAR(100),
    PRICE NUMBER,
    CATEGORY VARCHAR(20)
);
```

**과제**:
1. 각 테이블의 문제점 식별 (15점)
2. 표준화된 개선안 제시 (15점)

<details>
<summary>답안 예시</summary>

<b>1. 문제점 식별 (15점)</b><br><br>

<b>TBL_CUST 테이블:</b><br>
- 테이블명이 약어 사용 (TBL, CUST)<br>
- 컬럼명이 약어 중심 (C_ID, C_NM, REG_DT)<br>
- 일관성 없는 명명 (C_PHONE vs REG_DT)<br><br>

<b>ORDER_INFO 테이블:</b><br>
- 테이블명에 INFO 같은 모호한 용어 사용<br>
- 컬럼명 일관성 부족 (ORDER_NO vs ORD_DATE)<br>
- 외래키 명명이 다름 (CUSTOMER_ID vs C_ID)<br><br>

<b>PROD 테이블:</b><br>
- 테이블명이 약어 (PROD)<br>
- 컬럼명 일관성 부족 (PRODUCT_CODE vs PROD_NAME)<br><br>

<b>2. 표준화된 개선안 (15점)</b><br>

```sql
-- 표준화된 테이블들
CREATE TABLE 고객 (
    고객ID VARCHAR(10) PRIMARY KEY,
    고객명 VARCHAR(50) NOT NULL,
    전화번호 VARCHAR(15),
    가입일자 DATE
);

CREATE TABLE 주문 (
    주문번호 VARCHAR(20) PRIMARY KEY,
    고객ID VARCHAR(10) NOT NULL,
    주문일자 DATE,
    총주문금액 NUMBER,
    FOREIGN KEY (고객ID) REFERENCES 고객(고객ID)
);

CREATE TABLE 상품 (
    상품코드 VARCHAR(15) PRIMARY KEY,
    상품명 VARCHAR(100) NOT NULL,
    상품가격 NUMBER,
    상품카테고리 VARCHAR(20)
);

-- 표준 용어 사전 예시
용어: 고객ID
정의: 고객을 유일하게 식별하는 코드
형식: 영문 2자리 + 숫자 8자리
예시: CU12345678
```
</details>

**과제**: 스타 스키마 설계 (Fact 테이블 1개, Dimension 테이블 3개 이상)

<details>
<summary>답안 예시</summary>

<b>스타 스키마 설계:</b><br><br>

<b>Fact 테이블: 매출팩트</b><br>
```sql
CREATE TABLE 매출팩트 (
    매출팩트ID NUMBER PRIMARY KEY,
    일자키 NUMBER NOT NULL,
    상품키 NUMBER NOT NULL,

    FOREIGN KEY (상품키) REFERENCES 상품차원(상품키),
    FOREIGN KEY (고객키) REFERENCES 고객차원(고객키),
    FOREIGN KEY (시간키) REFERENCES 시간차원(시간키)
```

<b>Dimension 테이블들:</b><br>
```sql

    고객키 NUMBER PRIMARY KEY,
    고객ID VARCHAR(10),
    고객명 VARCHAR(50),
    성별 VARCHAR(1),
    연령대 VARCHAR(10),
    지역 VARCHAR(50),
    고객등급 VARCHAR(20)

-- 시간 차원
CREATE TABLE 시간차원 (
    시간키 NUMBER PRIMARY KEY,
    시간 NUMBER,
    시간대구분 VARCHAR(20), -- 새벽, 오전, 오후, 저녁
    업무시간여부 VARCHAR(1)
);
```
